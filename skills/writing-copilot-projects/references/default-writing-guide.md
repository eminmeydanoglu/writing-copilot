# Default Writing Guide

Use this as the baseline unless the project explicitly overrides it. Shared style files and the project's local style direction usually add or sharpen constraints; they should not be assumed to cancel this guide wholesale unless the project clearly says so.

## Baseline

- Prefer precise claims over inflated language.
- Prefer concrete nouns and verbs over abstraction stacks.
- Let each paragraph do one clear job.
- Preserve argumentative progression; do not jump between ideas for ornament.
- Use emphasis sparingly. Cut empty intensifiers, filler, and decorative repetition.
- Keep metaphors controlled; they should clarify, not fog the claim.
- Vary sentence length, but do not chase complexity for its own sake.
- Revise by delta. Improve the passage in front of you instead of rewriting the whole piece unless the task explicitly calls for a restructure.
- Preserve distinctive user phrasing when it carries voice.

## For Research-Backed Writing

- Separate direct support from interpretation.
- Do not attribute a claim to a source unless the source clearly supports it.
- Mark inference when the source is suggestive rather than explicit.

## For Revision

- Fix the bottleneck first: thesis, sequence, paragraph job, sentence clarity, then polish.
- Do not add flourish to cover conceptual weakness.
- When cutting, remove redundancy before removing specificity.
