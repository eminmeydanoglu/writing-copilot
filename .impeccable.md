## Design Context

- Target audience: Any AI user who wants to co-write longform text with agents.
- Primary use case: Keep a canonical markdown document stable while agents propose edits through a shadow file and the user reviews diffs before accepting them.
- Brand tone: Quiet, modern, editorial, tool-like rather than chatty.
- Product constraints:
  - Interface must stay simple.
  - Main surface is the document.
  - Diff view appears only when needed.
  - Agents should use normal file-editing tools against a shadow file instead of bespoke mutation tools.
